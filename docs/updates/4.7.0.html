Veldrid 4.7.0

2019-08-25

Veldrid 4.7.0 is now available for download. This is a small release with a few additions, and several miscellaneous bug fixes.

Veldrid can be installed from NuGet, and the source is available on GitHub.

As always, thanks to the contributors (see below) who made these improvements possible.

All Changes

  • [D3D11] workaround for emulated command lists when setting CB w/ offset. [e6647cf]
  • Guard glObjectLabel usage behind a null check. [2556ba2]
  • [OpenGL] Synchronize creation of Shaders and Pipelines. [4d09d45]
  • [OpenGL] Disable scissor test when calling glClear. [87a21f5]
  • [OpenGL] Fix UVW-STR sampler address mapping. [03fa437]
  • Fix mirror sampler address mode on metal [80aa5db] (@tzachshabtay)
  • Fix SamplerAddressMode.Clamp mapping in OpenGL [e32775f] (@tzachshabtay)
  • [Vulkan] Include UTF8 null terminator in more stack allocated string buffers. [c12e023]
  • Add BlendStateDescription.AlphaToCoverageEnabled. [4904770]
  • [Vulkan] More lenient surface extension logic. [85851ec]
  • Add more notes about Veldrid.SPIRV's handling of vertex semantics. [d1e5e92]
  • [OpenGL] Fix an issue with concurrent resource mapping. [e06b78c]
  • [OpenGL] Set scissor rect relative to framebuffer, not viewport. [cb08faf] (@TheMostCuriousThing)
  • [D3D11] Fix SRV memory leak. [c8f0515] (@TheMostCuriousThing)
  • [Metal] Fix mappings for SamplerDescription.AddressModeU/V/W. [be6d1db]
  • Add wrapper for SDL_SetWindowGrab. [c48646e]
  • [OpenGL] Ignore debug group messages in debug callback. [c85fd59]
  • Improve Vulkan memory allocation patterns and utilize dedicated allocation extensions. [3bcd470] [e5ddb93] [1ebe8c6]
  • Change so HandleMouseMotionEvent use relative mouse motion for delta. Update NeoDemo to use delta mouse movement for camera movement. [0c6ffb9] (@phelioz)
  • [OpenGL] Ignore UpdateTexture calls when region has 0 volume. [a1ba465]
  • Add wrapper for SDL_SetRelativeMouseMode and SDL_CaptureMouse. Add comments for existing Sdl2.Mouse methods. [d718391] (@phelioz)
  • Add wrapper for SDL_JoystickInstanceID [d0d0f4d] (@phelioz)
